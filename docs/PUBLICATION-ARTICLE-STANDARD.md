# Publication Article Standard — LevNytt V1.0

**Date:** 2026-06-30
**Status:** FROZEN — single source of truth for all LevNytt informational articles
**Scope:** All informational articles published to `https://levnytt.se/`

---

## Purpose

This document defines the **canonical template** for every informational article published on LevNytt.

The Publication Agent (`.opencode/agents/publication-agent.md`) deploys articles from `content/articles/` to the production root. Every article that passes through the Publication Agent pipeline conforms to this standard.

### Single Source of Truth

- This document **is** the article standard.
- `levnytt-se-master-template.html` is **legacy** — it does not describe current articles.
- Any discrepancy between an article on disk and this document means the **article is wrong**.
- This document controls the template. The PA articles on disk are **output** — this document is the **specification**.

### What this standard covers

Informational articles: health, wellness, supplement science, consumer education. These are the articles generated by the publication pipeline and listed in `docs/editorial-backlog/production-status.md`.

### What this standard does NOT cover

- Pillar/authority pages (historia, vetenskap, hallbarhet, etc.)
- Product pages (neolife-*.html)
- The homepage (index.html)
- Utility/tool pages (calculator)
- Legal pages (integritetspolicy)

Those pages use separate template families and are documented in `docs/ARTICLE-TEMPLATE-FAMILIES.md`.

### Migration Principle

> **Legacy informational articles are migrated to match this standard. The Publication Agent template is the single source of truth. Legacy templates are not preserved.**
>
> When an article is migrated, its existing HTML is restructured to match this standard exactly. Old layout patterns (gradient hero, breadcrumbs bar, authority-nav, journey blocks, gold conclusions, freshness banners, etc.) are removed — not preserved, not archived as alternate layouts.
>
> After migration, the article is structurally indistinguishable from an article generated fresh by the Publication Agent pipeline.

### Content Preservation Principle

During migration, the presentation layer is updated. Editorial content is never rewritten.

**Preserved during migration:**
- All editorial content (paragraphs, lists, tables, statistics)
- All headings (unless they violate the standard's heading format)
- The article URL (slug is never changed)
- SEO intent (title and description meaning, not necessarily exact wording)
- All references and citations (PubMed links, source attributions)
- All internal links to other LevNytt pages (preserved or relocated to tierbox/CTA)

**Not preserved during migration:**
- Layout and CSS (replaced by standard ia-wrap structure)
- Component ordering (reordered to match PA standard)
- Legacy class names (replaced by standard ia-* classes)
- Navigation and footer markup (replaced by shared nav.js / footer.js)
- Schema format (converted to @graph Article + FAQPage)
- Font declarations (standardized to Playfair Display + Inter)

**This is a template migration, not an editorial rewrite.** The words stay. The HTML changes.

### Definition of Done

A legacy informational article is considered successfully migrated when it passes all of the following checks:

| # | Check | Validation Method |
|---|---|---|
| 1 | Matches the Publication Article Standard structurally | Visual diff against the skeleton in the Appendix — structural element count and order must match |
| 2 | All mandatory components are present | Count: doctype, charset, viewport, title, description, canonical, verification metas, OG metas, twitter metas, JSON-LD @graph, fonts, pillar.css, inline ia-* CSS, #site-nav + nav.js, ia-wrap + article, category label, eyebrow, h1, byline, punchline, Key Takeaways, FAQ, method note, tierbox, CTA, disclosure, author box, final disclosure, footer.js, components.js |
| 3 | No forbidden components remain | Search for: freshness-banner, hero-overlay, hero-stats, back-link-wrapper, breadcrumbs (HTML), authority-nav, journey-block, tier-section, callout-gold, cta-block, article-wrap, article-content, article-inner, toc, stats-row, author-box (legacy), faq-section, faq-body, itemscope in body, inline nav, inline footer CSS, smooth scroll script |
| 4 | No legacy template structures remain | No gradient hero, no HTML breadcrumbs bar, no authority nav, no journey block, no gold conclusion, no TOC, no sources block |
| 5 | No duplicate ia-* CSS remains | All ia-* classes match the exact CSS values specified in section 7 |
| 6 | No legacy JavaScript remains | No inline smooth scroll, no duplicate nav.js at bottom, no inline nav injection |
| 7 | Metadata complies with the standard | Title format, description length, canonical URL, OG meta completeness, twitter cards, verification codes |
| 8 | JSON-LD complies with the standard | Single @graph block, Article + FAQPage, author @type Person, valid dates, all FAQ questions included |
| 9 | Navigation and footer are loaded through shared components | nav.js at top with #site-nav div, footer.js at bottom, components.js at bottom, all with defer |
| 10 | Visual appearance matches a current PA article | Eyeball test: the page looks like a PA article, not a legacy article |
| 11 | Article validates without HTML errors | Run through an HTML validator, zero errors |
| 12 | Editorial content is preserved | All original paragraphs, lists, statistics, citations, and internal links remain |

When all 12 checks pass, the article status is updated to `**Migrated**` in the migration tracking document.

## Required Components

Every informational article MUST contain the following components in this exact order.

Components marked **Mandatory** produce a validation error if missing.
Components marked **Optional** may be omitted when not relevant to the article topic.

### 1. Document Head

**Mandatory**

```html
<!DOCTYPE html>
<html lang="sv">
<head>
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

**Purpose:** Standard HTML5 document shell.

### 2. SEO Metadata

**Mandatory**

```html
<title>Article Topic — descriptive suffix | LevNytt</title>
<meta name="description" content="155-character max description of the article.">
<link rel="canonical" href="https://levnytt.se/article-slug">
```

**Rules:**
- Title format: `Topic — descriptive suffix | LevNytt`
- Description: 150–155 characters. One sentence describing the article's value.
- Canonical: absolute URL with `https://` and `levnytt.se`

**Additional meta tags (all Mandatory):**

```html
<meta name="google-site-verification" content="kAcoLDFGCpGh42gIFRgPeWlC253vTP3OLBs6wI8KDQ0">
<meta name="p:domain_verify" content="6a9e88f7014abe0735767f464c08f337"/>
```

**Robots meta (Recommended):**

```html
<meta name="robots" content="index, follow, max-snippet:-1, max-video-preview:-1, max-image-preview:large">
```

### 3. Open Graph Metadata

**Mandatory**

```html
<meta property="og:type" content="article">
<meta property="og:title" content="Full article title (can differ slightly from <title>)">
<meta property="og:description" content="Social-share description">
<meta property="og:url" content="https://levnytt.se/article-slug">
<meta property="og:site_name" content="LevNytt">
<meta property="og:locale" content="sv_SE">
<meta property="og:image" content="https://levnytt.se/images/og/default.jpg">
<meta property="og:image:width" content="1200">
<meta property="og:image:height" content="630">
<meta property="og:image:alt" content="LevNytt — En oberoende NeoLife-distributör">
```

**Rules:**
- `og:type` must be `article` (never `website` for informational articles)
- `og:image` must be the absolute URL to the default OG image
- All three image dimensions (width, height, alt) must be present

### 5. JSON-LD Structured Data

**Mandatory**

Every article must contain a single `@graph` JSON-LD block with `Article` and `FAQPage` types.

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@graph": [
    {
      "@type": "Article",
      "headline": "Full article title",
      "description": "Full description",
      "url": "https://levnytt.se/article-slug",
      "datePublished": "2026-06-30",
      "dateModified": "2026-06-30",
      "author": {
        "@type": "Person",
        "name": "Author Name",
        "url": "https://levnytt.se/author-page"
      },
      "publisher": {
        "@type": "Organization",
        "name": "LevNytt",
        "url": "https://levnytt.se"
      },
      "mainEntityOfPage": {
        "@type": "WebPage",
        "@id": "https://levnytt.se/article-slug"
      }
    },
    {
      "@type": "FAQPage",
      "mainEntity": [
        {
          "@type": "Question",
          "name": "Question text?",
          "acceptedAnswer": {
            "@type": "Answer",
            "text": "Answer text."
          }
        }
      ]
    }
  ]
}
</script>
```

**Rules:**
- Single `<script>` block containing one `@graph` array
- Always includes `Article` + `FAQPage`
- `author` must be `@type: Person` (never `Organization`)
- `datePublished` and `dateModified` must be ISO 8601 dates (YYYY-MM-DD)
- All 8+ FAQ entries must be included (minimum 6 required)
- BreadcrumbList is **not** included in the @graph. PA articles do not use BreadcrumbList schema.
- Pinterest rich pin meta is optional: `<meta name="pinterest-rich-pin" content="enabled">` + `pinterest:description`

### 6. Fonts

**Mandatory**

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Playfair+Display:wght@400;600;700&family=Inter:wght@300;400;500;600&display=swap" rel="stylesheet">
```

**Rules:**
- Playfair Display: 400, 600, 700 (never 800)
- Inter: 300, 400, 500, 600 (never 700 or 800)
- Both preconnect tags must precede the font link
- No other font families permitted for informational articles

### 8. Navigation

**Mandatory**

```html
<body>
<div id="site-nav"></div>
<script src="/nav.js" defer></script>
```

**Rules:**
- The `<div id="site-nav">` placeholder is required before the nav.js script
- `nav.js` must be loaded with `defer`
- This is the first content inside `<body>`
- No inline `<nav>` element (nav.js handles injection)
- No hardcoded navigation HTML

### 9. Content Wrapper

**Mandatory**

```html
<div class="ia-wrap">
<article>
```

**Rules:**
- `.ia-wrap` is the outermost content container
- `<article>` wraps all article content
- Do NOT use `<main class="ia-wrap">` — use `<div class="ia-wrap"><article>`
- Do NOT use `.article-wrap`, `.article-content`, `.article-inner`, or `.container` classes

**Mandatory**

```html
<p style="font-size:0.85em;color:#888;text-transform:uppercase;letter-spacing:0.05em">Topic Area · Subcategory</p>
```

**Purpose:** SEO breadcrumb visible to search engines. Provides topical context.

**Format:** `Primary Topic · Secondary Descriptor` (e.g., "Magnesium · Stress", "Vitamin D · Dosering")

**Mandatory**

```html
<div class="ia-eyebrow">Kosttillskott · Forskning</div>
```

**Purpose:** Visual category tag at article top. CSS rendered from pillar.css.

**Rules:**
- Short, 2–3 word category classification
- Common values: `Kosttillskott · Forskning`, `Näringslära · Evidens`, `Hälsa · Konsument`

### 12. H1 Title

**Mandatory**

```html
<h1>Full Article Title — uses em-dash for subtitles</h1>
```

**Rules:**
- Single `<h1>` per article
- Styled by `.ia-wrap h1` in inline CSS
- Use em-dash (—) for subtitles, not colon
- Title must match the JSON-LD `headline` field

### 13. Author Byline

**Mandatory**

```html
<p>Av <a target="_blank" rel="noopener noreferrer" href="https://levnytt.se/den-fundersamma-mannen">Den Fundersamma Mannen</a> · Senast uppdaterad: juni 2026 · 7 min läsning</p>
```

**Rules:**
- Single `<p>` tag, not `<div>`
- Author name linked to author page
- Date format: `månad år` (month in Swedish, year)
- Reading time: `X min läsning`
- Separated by middle dots (·)
- Do NOT use `<div class="ia-meta">` with `<span class="ia-meta-author">` — the canonical format is a plain `<p>` with middle dots

### 14. Punchline

**Mandatory**

```html
<p class="ia-punchline">The central claim of the article — one clear, evidence-backed statement that answers the reader's question directly. Bold text wraps the key assertion.</p>
```

**Purpose:** The article's value proposition. Answers "what will I learn?" in one paragraph.

**Rules:**
- Single paragraph
- Contains the core answer the article provides
- Typically uses `<strong>` on the key claim
- Styled with gold left border and cream background

- Dark green background (#1B4332) with cream text

### 16. Main Content Sections

**Mandatory**

```html
<h2>Section heading — a question or declarative statement</h2>
<p>Paragraph content. References to studies are linked inline: (<a href="https://pubmed.ncbi.nlm.nih.gov/...">Author et al., Journal Year</a>). <span class="ia-evidence-label ia-ev-t2">Tier 2</span></p>
<p>Second paragraph continuing the section.</p>

<h3>Sub-section heading</h3>
<p>Content under h3.</p>
```

**Rules:**
- Use `<h2>` for major sections (question-format encouraged)
- Use `<h3>` for sub-sections within an h2
- Every factual health claim gets an evidence label
- References linked directly to PubMed or official sources
- Internal links to other LevNytt articles are encouraged but not required in every section
- No `<h4>` or deeper nesting in content body

### 17. Evidence Labels

**Mandatory** on all factual health claims

```html
<span class="ia-evidence-label ia-ev-t1">Tier 1</span>
<span class="ia-evidence-label ia-ev-t2">Tier 2</span>
<span class="ia-evidence-label ia-ev-t3">Tier 3</span>
<span class="ia-evidence-label ia-ev-t4">Tier 4</span>
```

**Tier definitions:**
- Tier 1: Systematic review, meta-analysis, or official health authority (Livsmedelsverket, EFSA, WHO)
- Tier 2: Individual RCT or well-designed cohort study
- Tier 3: Observational study, cross-sectional, or smaller clinical trial
- Tier 4: Mechanistic research, expert opinion, theoretical basis

**Rules:**
- Every factual health claim in Key Takeaways and body content gets a tier badge
- Badge appears inline, immediately after the claim
- Color coding: Tier 1 (green), Tier 2 (blue), Tier 3 (yellow), Tier 4 (red)

- Tier 2 typically links to the relevant "komplett guide"

```html
</details>

<details><summary>Second question?</summary>
<p>Second answer with a link to a relevant page: <a target="_blank" rel="noopener noreferrer" href="https://levnytt.se/related">Related article</a>.</p>
</details>
</div>
```

**Rules:**
- Heading: `Vanliga frågor om [article topic]`
- Native `<details>/<summary>` accordion (no JavaScript)
- 6–10 question/answer pairs
- Questions match the FAQPage schema in JSON-LD
- Answers are 2–4 sentence paragraphs
- Internal links to other LevNytt articles are encouraged
- Do NOT use `itemscope` microdata in body (FAQ schema is in JSON-LD head only)
- Do NOT wrap answers in `<div class="faq-body">` or `<div class="faq-answer">`

**Mandatory**

```html
<p class="ia-disclosure"><strong>Metodnotering:</strong> YMYL-classified article. Methodology notes, evidence tier explanation, related reading links. <a target="_blank" rel="noopener noreferrer" href="https://levnytt.se/related">Related article link</a>.</p>
```

**Rules:**
- Border-top separator line
- Small text (0.82em)
- Includes methodology context
- Links to 1–3 related articles

### 28. Closing Tags and Scripts

**Mandatory**

```html

</div>
```

- Do NOT include an inline smooth scroll script (pillar.css provides `scroll-behavior:smooth`)
- No hardcoded footer HTML (footer.js handles injection)

## Legacy Components (Do Not Use)

The following components appear in legacy articles and pillar/product pages. They are **forbidden** in Publication Agent informational articles. These must be removed during migration — never preserved.

### Forbidden Components

| Component | Class/Element | Reason |
|---|---|---|
| Freshness banner | `<div class="freshness-banner">` | PA template does not use it. Content freshness is communicated via byline date. |
| Gradient hero | `<section class="hero">` with `.hero-overlay` | PA template uses flat header (eyebrow + h1 + byline). |
| Hero stats | `<div class="hero-stats">` | PA template uses `.ia-stat-grid` inline in content. |
| Back link | `<div class="back-link-wrapper">` | PA template has no back-link. Navigation is via nav.js. |
| HTML breadcrumbs bar | `<div class="breadcrumbs">` | PA template includes no visible breadcrumbs. Topic context comes from category label. |
| Authority navigation | `<div class="authority-nav">` | PA template has no authority nav. Internal links use `.ia-tierbox`. |
| Journey block | `<div class="journey-block">` | PA template has no reader journey navigation. |
| Tier section (8-link) | `<div class="tier-section">` with `.tier-grid` | PA template uses `.ia-tierbox` with exactly 4 links. |
| Gold conclusion callout | `<div class="callout callout-gold">` with `<h4>Slutsats</h4>` | PA template has no conclusion callout. |
| Summary callout | `<div class="callout">` with `<h4>Sammanfattning</h4>` | PA template uses `.ia-punchline` + `.ia-takeaways` instead. |
| TOC | `<div class="toc">` | PA template does not include a table of contents. |
| Article-wrap structure | `<main class="article-wrap"><div class="container"><div class="article-body">` | PA template uses `<div class="ia-wrap"><article>`. |
| Article-content structure | `<div class="article-content"><div class="article-inner">` | PA template uses `<div class="ia-wrap"><article>`. |

| Itemscope FAQ microdata | `itemscope itemprop="mainEntity"` in HTML body | PA template puts FAQ schema in JSON-LD head only. Body uses `<details>/<summary>`. |
| Inline nav element | `<nav>`, `.nav-inner`, `.nav-brand`, `.nav-links` | PA template uses `#site-nav` div + nav.js. |
| Inline footer CSS | Footer grid, `.footer-grid`, `.footer-col` | PA template uses footer.js only. |
| Duplicate ia-* CSS | Any ia-* class declared differently from the standard | Exact CSS values in this document are canonical. |
| Smooth scroll script | `<script>document.querySelectorAll('a[href^="#"]')...` | pillar.css provides `scroll-behavior: smooth`. |
| Sources block | `<p style="border-top..."><strong>Källor:</strong>` | PA template uses `.ia-disclosure` and inline references. |
| Last updated paragraph | `<p class="last-updated">` | PA template includes date in author byline and disclosure. |
| Inline :root tokens | `:root{--gold:#C8962E;...}` | pillar.css provides all design tokens. |
| `og:type: "website"` | Used in S3 trust pages | Must be `"article"` for informational articles. |
| `WebPage` schema type | Used in S3 trust pages | Must be `Article` for informational articles. |
